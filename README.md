# LogFood

Учебно-практическое React-приложение для учета питания по дням. LogFood помогает вести дневник еды, считать калории за выбранную дату и хранит данные локально в браузере.

Текущий этап проекта: дневник считает итоговые калории по весу продукта и калорийности на 100 г, показывает дневную цель калорий, прогресс к ней через donut-диаграмму, недельную статистику и помогает быстро вводить продукты через локальный справочник.

## Возможности

- добавление записи еды по названию, весу и ккал на 100 г;
- локальный справочник продуктов;
- подсказки продуктов при вводе названия еды;
- выбор продукта из подсказок с автозаполнением названия и ккал на 100 г;
- быстрый ввод вроде `гречка 100 г` с автоматическим заполнением веса;
- поиск подсказок по очищенному названию из быстрого ввода;
- сохранение записи с названием без веса из быстрого ввода;
- автоматический расчет итоговых калорий по формуле `grams * caloriesPer100g / 100`;
- валидация формы добавления;
- редактирование записи с пересчетом калорий;
- подтверждение удаления записи;
- выбор даты дневника;
- просмотр записей за конкретный день;
- история дней с переходом на выбранную дату;
- подсчет общей суммы калорий за день;
- дневная цель калорий с сохранением в `localStorage`;
- donut-диаграмма прогресса: съедено, осталось, цель выполнена или превышена;
- отдельная страница статистики за неделю;
- недельные итоги: сумма, среднее за день и количество дней с превышением цели;
- progress bar по каждому дню недели;
- сохранение записей в `localStorage`;
- защита от ошибок чтения и записи `localStorage`;
- маршруты `/`, `/day/:date`, `/history`, `/statistics`;
- fallback route для неизвестных адресов.

## Модель записи

```ts
type FoodEntry = {
  id: number;
  title: string;
  grams: number;
  caloriesPer100g: number;
  calories: number;
  date: string;
  createdAt: string;
};
```

`calories` хранится как рассчитанное значение, чтобы список и дневная сумма не пересчитывали формулу в UI.

## Основные сценарии

- открыть дневник;
- выбрать дату;
- добавить запись еды с весом и ккал на 100 г;
- проверить рассчитанные итоговые калории;
- исправить вес или ккал на 100 г;
- проверить пересчет записи и дневной суммы;
- изменить дневную цель калорий;
- проверить прогресс, остаток и превышение цели в donut-диаграмме;
- отменить редактирование;
- удалить запись через подтверждение;
- перейти в статистику;
- проверить недельную сумму, среднее значение и прогресс по дням;
- перейти в историю;
- открыть день из истории;
- обновить страницу и проверить, что записи сохранились.
